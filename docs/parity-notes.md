## Unintentional differences (fixed in this task)

### FIXED: `{{NAME_UPPER}}` not substituted in `modules/mcp-json.tpl`

**Symptom (before fix):** Generated `.mcp.json` contained `${ATLASSIAN__TOKEN}` (double
underscore) for every Atlassian workspace instead of e.g. `${ATLASSIAN_CENCOSUD_TOKEN}`.

**Root cause:** The `mcp-json.tpl` template used `{{NAME_UPPER}}` inside the
`{{#each MCPS_ATLASSIAN}}` block:
```
"CONFLUENCE_API_TOKEN": "${ATLASSIAN_{{NAME_UPPER}}_TOKEN}",
```

The render engine's `_render_each` function, for a field named `name` with value `cencosud`,
substitutes:
- `{{name}}` → `cencosud`  (lowercase key → lowercase value)
- `{{NAME}}` → `CENCOSUD`  (uppercase key → uppercase value)

It does NOT create a `NAME_UPPER` variable — `NAME_UPPER` is never set, so it resolves to
empty string, yielding `${ATLASSIAN__TOKEN}`.

**Fix applied:** Changed `{{NAME_UPPER}}` to `{{NAME}}` in `modules/mcp-json.tpl` (lines
25 and 28). Now generates `${ATLASSIAN_CENCOSUD_TOKEN}` and `${ATLASSIAN_PERSONAL_TOKEN}`
correctly.

**Test coverage added:** `tests/mcp-json.bats` test 4 now asserts
`CONFLUENCE_API_TOKEN` and `JIRA_API_TOKEN` values to prevent regression.
