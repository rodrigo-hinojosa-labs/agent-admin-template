## Intentional differences

### CLAUDE.md

The template generates an English-language, generic `CLAUDE.md`. The reference
agent's version is Spanish with heavy personal context (infra tables, smart-home
devices, contacts, calendars, specific Telegram chat IDs, SSH commands, inherited
role definitions, service management commands). These sections are not backed by
template variables — they are user-edited content added after initial generation,
as intended by the design.

Specifically absent from the template output (by design):
- `## Este Repo` intro section
- `## Infraestructura` (RPi cluster table, IP addresses, SSH)
- `## Servicios` (per-host Docker services)
- `## MCP Servers Disponibles` (global MCP path note)
- `## Smart Home` (smart-home device inventory)
- `## Contactos` + `### Calendarios`
- `## Notificaciones` (hardcoded Chat ID, specific token var name)
- `## Permisos Pre-aprobados` (SSH allow-list)
- `## Plugins` table (with descriptions)
- `## Equipo de Agentes` + `## Roles Heredados` (references to other custom agents)
- `## Gestión del Servicio` (systemctl/launchctl one-liners)
- Heartbeat section is in English in template; Spanish + more detailed in the reference agent's version.
  Also, the reference agent's version includes `## Comando /heartbeat (Telegram)` (added after generation).

Language: template uses English headings and principles; the reference agent's uses Spanish (by design).

### .mcp.json

Template defaults include `fetch`, `time`, `sequential-thinking` (in addition to
`playwright`). The `the reference agent/.mcp.json.example` only has `playwright` plus
Atlassian — this is because that example was created manually before the template existed,
not generated by the template. The template output is correct.

Template includes GitHub MCP (`enabled: true` in fixture); the reference agent's example does not.
Both differences are intentional — the fixture reflects the desired future state.

Atlassian workspace name difference: fixture uses two sample workspaces
(`work` and `personal`) with placeholder URLs; the reference hand-built
agent used a different set of workspace aliases. Intentional, not a
template bug.

### heartbeat.conf

- the reference agent's `HEARTBEAT_INTERVAL` is `"1m"` (overridden by `launch.sh run` at runtime);
  template generates `"30m"` (the fixture default). This is expected — the reference
  agent's conf was modified after initial generation.
- the reference agent's `HEARTBEAT_PROMPT` is a very specific project-tracker + host status report; template
  generates `"Check status and report"`. Intentional — user customized it after generation.
- the reference agent's file has human-readable inline comments per-variable; template's generated file

  The `heartbeat.sh` script in the reference agent's repo reads this from the channel config, not the
  conf file — minor structural divergence, not breaking.

## Unintentional differences (fixed in this task)

### FIXED: `{{NAME_UPPER}}` not substituted in `modules/mcp-json.tpl`

**Symptom (before fix):** Generated `.mcp.json` contained `${ATLASSIAN__TOKEN}` (double
underscore) for every Atlassian workspace instead of e.g. `${ATLASSIAN_WORK_TOKEN}`.

**Root cause:** The `mcp-json.tpl` template used `{{NAME_UPPER}}` inside the
`{{#each MCPS_ATLASSIAN}}` block:
```
"CONFLUENCE_API_TOKEN": "${ATLASSIAN_{{NAME_UPPER}}_TOKEN}",
```

The render engine's `_render_each` function, for a field named `name` with value `work`,
substitutes:
- `{{name}}` → `work`  (lowercase key → lowercase value)
- `{{NAME}}` → `WORK`  (uppercase key → uppercase value)

It does NOT create a `NAME_UPPER` variable — `NAME_UPPER` is never set, so it resolves to
empty string, yielding `${ATLASSIAN__TOKEN}`.

**Fix applied:** Changed `{{NAME_UPPER}}` to `{{NAME}}` in `modules/mcp-json.tpl` (lines
25 and 28). Now generates `${ATLASSIAN_WORK_TOKEN}` and `${ATLASSIAN_PERSONAL_TOKEN}`
correctly.

**Test coverage added:** `tests/mcp-json.bats` test 4 now asserts
`CONFLUENCE_API_TOKEN` and `JIRA_API_TOKEN` values to prevent regression.
